# Task 017: xy run mode

## Objective

Implement dkg.modes.xy.run() for cross-matrix predictor-to-target analysis, producing all three tier Parquet files plus a Tier 0 marginals file.

## Acceptance Criteria

- [ ] `run(config)` produces tier0_marginals.parquet, tier1_screen.parquet, tier2_deep.parquet, tier3_stability.parquet in config.output_dir
- [ ] tier0_marginals.parquet contains Phase 1 profiles for all X and Y columns with a source column ("x" or "y")
- [ ] All output files are non-empty Parquet files readable by polars
- [ ] Clear error raised if x_matrix_path or y_matrix_path is None
- [ ] All default checks pass

## Inputs

- src/dkg/io.py
- src/dkg/tier1.py
- src/dkg/tier2.py
- src/dkg/tier3.py
- src/dkg/phases/phase1.py

## Outputs

- src/dkg/modes/xy.py
- tests/test_mode_xy.py

## Requirements

- Load X and Y matrices via load_matrix(); align via align_matrices()
- Compute Phase 1 marginal profiles for all X columns and all Y columns via sweep_phase1(); combine with a source column and write to tier0_marginals.parquet
- Call tier1.screen(X, Y, config) in xy mode; write tier1_screen.parquet
- Call tier2.run_deep() passing precomputed Phase 1 profiles; write tier2_deep.parquet
- Call tier3.run_stability(); write tier3_stability.parquet
- Warn (do not error) if X and Y column namespaces overlap
