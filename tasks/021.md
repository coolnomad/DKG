# Task 021: Numerical validation test suite

## Objective

Add tests/test_phase_parity.py that runs each Python phase function against the R reference implementation on matched inputs and asserts numerical equivalence within documented tolerances.

## Acceptance Criteria

- [ ] Tests for all phases 1-9 present in test_phase_parity.py
- [ ] Each test compares all numeric output fields between Python and R within documented tolerance
- [ ] Tests are automatically skipped if Rscript not on PATH or --depmap-path not provided
- [ ] At least one test runs unconditionally on synthetic data without requiring R or real data
- [ ] Tolerances for CV-based phases (9) documented with rationale
- [ ] All default checks pass

## Requirements

- Use pytest --depmap-path CLI option via conftest.py to pass path to real DepMap data
- For each phase, extract PPARG and RXRA_RXRB columns from real data; run Python phase function; run R equivalent via Rscript subprocess; compare all numeric fields
- Default tolerance atol=1e-6 for deterministic phases (1-8); document per-field exceptions
- Phase 9 uses atol=1e-3 due to fold-assignment RNG differences between R and Python; document this explicitly in a comment
- Skip marker: pytest.mark.skipif(shutil.which("Rscript") is None or depmap_path is None)
- Unconditional synthetic test: generate two random vectors (n=50, seed=42); run Python phase 2; assert output has expected fields and all values are finite
