# Task 020: CLI entry point

## Objective

Implement dkg.cli and dkg.__main__ using argparse so the package is fully runnable from the command line and dispatches to the correct run mode.

## Acceptance Criteria

- [ ] `python -m dkg --help` exits 0 and lists all flags
- [ ] `python -m dkg --version` exits 0 and prints the package version
- [ ] `python -m dkg --mode xy --x-matrix X.csv --y-matrix Y.csv --output-dir out` dispatches to modes.xy.run()
- [ ] `--config-json path` loads a JSON file and merges into RunConfig before CLI flags are applied
- [ ] Invalid mode exits 1 with a clear error message to stderr
- [ ] All default checks pass

## Inputs

- src/dkg/config.py
- src/dkg/modes/xy.py
- src/dkg/modes/xx.py
- src/dkg/modes/pair.py
- src/dkg/__main__.py

## Outputs

- src/dkg/cli.py
- src/dkg/__main__.py
- tests/test_cli.py

## Requirements

- Top-level flags: --mode {xy,xx,pair}, --x-matrix, --y-matrix (optional for xx mode), --output-dir, --config-json, --n-jobs, --tier1-threshold, --top-k, --graph-edge-threshold, --pair-x, --pair-y
- Parse flags into RunConfig; --config-json loads JSON and sets RunConfig fields before CLI flag overrides are applied
- Dispatch to modes.xy.run(), modes.xx.run(), or modes.pair.run() based on mode
- Print one-line completion summary and output directory path on success
- --version reads from importlib.metadata.version("dkg")
- Exit 0 on success, 1 on any error with message to stderr
