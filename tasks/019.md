# Task 019: pair run mode (R subprocess delegation)

## Objective

Implement dkg.modes.pair.run() for single-pair deep-dive by delegating all phase computation to fitting_functions.R via subprocess.

## Acceptance Criteria

- [ ] `run(config)` with valid pair_x and pair_y produces pair_result.parquet in config.output_dir
- [ ] Preflight check raises a clear error with remediation instructions if Rscript not on PATH
- [ ] Preflight check raises a clear error if fitting_functions.R not found at expected path
- [ ] pair_result.parquet is readable by polars and contains at minimum phase 2 metrics
- [ ] All default checks pass

## Inputs

- src/dkg/io.py
- fitting_functions.R

## See Also

- fitting_functions.R

## Outputs

- src/dkg/modes/pair.py
- run_pair.R
- tests/test_mode_pair.py

## Requirements

- Accept config.pair_x and config.pair_y column names; load from config.x_matrix_path and config.y_matrix_path respectively; align on shared rows
- Write aligned vectors to a temporary CSV file
- Create run_pair.R wrapper that sources fitting_functions.R, reads the CSV, runs all phases (1-10), and writes results as CSV to a temp output file; path to temp output file passed as command-line argument
- Call via subprocess.run(["Rscript", "run_pair.R", input_csv, output_csv], capture_output=True, text=True)
- Parse CSV output into a Python dict and write to pair_result.parquet
- Preflight: check shutil.which("Rscript") is not None and Path("fitting_functions.R").exists() before running; raise RuntimeError with clear message and remediation steps if either check fails
- Tests may be skipped automatically if Rscript not on PATH via pytest.mark.skipif
