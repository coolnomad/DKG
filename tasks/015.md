# Task 015: Tier 2 pipeline orchestrator

## Objective

Implement dkg.tier2.run_deep() which filters Tier 1 pairs by threshold, runs phases 2-9 on each surviving pair in parallel, and writes tier2_deep.parquet.

## Acceptance Criteria

- [ ] `run_deep(tier1_df, X, Y, x_cols, y_cols, phase1_x, phase1_y, config)` returns a Polars DataFrame
- [ ] Output has x_col and y_col columns prepended; all phase 2-9 output columns follow
- [ ] Pairs where a phase raises are recorded with NaN for that phase's columns; run continues
- [ ] tier2_deep.parquet is written to config.output_dir
- [ ] joblib parallelism used with config.n_jobs
- [ ] All default checks pass

## Inputs

- src/dkg/tier1.py

- src/dkg/phases/phase3.py
- src/dkg/phases/phase4.py

- src/dkg/phases/phase6.py
- src/dkg/phases/phase7.py
- src/dkg/phases/phase8.py
- src/dkg/phases/phase9.py

## Outputs

- src/dkg/tier2.py
- tests/test_tier2.py

## Requirements

- Filter tier1_df to rows where passes_threshold(row, config) is True
- Accept precomputed Phase 1 marginal profiles for all X and Y columns as phase1_x and phase1_y Polars DataFrames; do not recompute Phase 1 per pair
- For each surviving pair run phases 2-9 sequentially; catch per-phase exceptions, fill NaN for that phase's columns, log a warning via Python logging
- Assemble per-pair flat dicts into a Polars DataFrame; write to tier2_deep.parquet
- Use joblib.Parallel(n_jobs=config.n_jobs, backend="loky") for pair-level parallelism
