# Task 004: Tier 1 vectorized correlation screen

## Objective

Implement dkg.tier1.screen() using fully vectorized matrix operations to compute Pearson and Spearman correlation for all pairs without Python loops. Write results to tier1_screen.parquet.

## Acceptance Criteria

- [ ] `screen(X, Y, config)` produces a Polars DataFrame with columns x_col, y_col, pearson_r, pearson_p, spearman_r, spearman_p, n_obs
- [ ] Pearson r values match scipy.stats.pearsonr within atol=1e-10 on a 50-column test case
- [ ] Spearman r values match scipy.stats.spearmanr within atol=1e-10 on a 50-column test case
- [ ] xx mode produces only upper-triangle pairs (no duplicate pairs, no self-pairs)
- [ ] Output is written to tier1_screen.parquet in config.output_dir
- [ ] All default checks pass

## Inputs

- src/dkg/tier1.py
- src/dkg/config.py

## Outputs

- src/dkg/tier1.py
- tests/test_tier1.py

## Requirements

- Pearson r via mean-centered dot products: center each column, then (X_c.T @ Y_c) / (n * std_x * std_y)
- Spearman r by rank-transforming each column then reusing the Pearson path
- Two-sided p-values from t-distribution: t = r * sqrt((n-2)/(1-r²)), p = 2 * scipy.stats.t.sf(abs(t), df=n-2)
- xy mode: all n_x x n_y pairs
- xx mode: upper triangle only (i < j)
- Pairwise complete-observation masking: if any NA present in a column pair, compute n_obs for that pair and use only complete rows
- If RAM pressure is prohibitive at full scale, chunk column blocks and accumulate; document chunking strategy and memory estimate in a module-level comment
- `passes_threshold(row, config) -> bool`: returns True if abs(pearson_r) >= config.tier1_pearson_threshold OR abs(spearman_r) >= config.tier1_spearman_threshold
