# Task 003: Matrix I/O module

## Objective

Implement dkg.io with functions to load numeric matrices from disk and align two matrices on their shared observation index.

## Acceptance Criteria

- [ ] `load_matrix("file.csv")` returns a tuple of (float64 ndarray, row_labels list, col_labels list)
- [ ] .feather and .arrow inputs load correctly via polars
- [ ] .csv.gz loads correctly
- [ ] `align_matrices(X, X_rows, Y, Y_rows)` inner-joins on shared row labels and returns aligned arrays
- [ ] align_matrices raises ValueError with a clear message if fewer than 10 observations survive alignment
- [ ] All-NaN matrix raises ValueError
- [ ] All default checks pass

## Inputs

- src/dkg/io.py
- src/dkg/config.py

## Outputs

- src/dkg/io.py
- tests/test_io.py

## Requirements

- `load_matrix(path: str) -> tuple[np.ndarray, list[str], list[str]]` returns (data, row_labels, col_labels)
- Route to polars for .feather/.arrow, numpy/polars for .csv/.csv.gz
- All data cast to float64; missing values become NaN; no imputation
- `align_matrices(X: np.ndarray, X_rows: list[str], Y: np.ndarray, Y_rows: list[str]) -> tuple[np.ndarray, np.ndarray, list[str]]` inner-joins on shared row labels
- Raise ValueError if fewer than 10 shared rows survive alignment
- Raise ValueError if any matrix is all-NaN
- Raise ValueError if column count < 1
- Tests cover: CSV round-trip, feather round-trip, alignment with partial overlap, alignment failure below 10 rows, all-NaN rejection
