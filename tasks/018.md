# Task 018: xx run mode and graph construction

## Objective

Implement dkg.modes.xx.run() for symmetric within-matrix analysis including Louvain community detection and graph output.

## Acceptance Criteria

- [ ] `run(config)` produces tier0_marginals.parquet, tier1_screen.parquet, tier2_deep.parquet, tier3_stability.parquet, graph.graphml, and communities.parquet
- [ ] Tier 1 output contains only upper-triangle pairs (no duplicates, no self-pairs)
- [ ] graph.graphml is a valid networkx-readable file
- [ ] communities.parquet has columns: node, community_id, degree
- [ ] Known strongly-connected nodes in a small synthetic test graph appear in the same community
- [ ] All default checks pass

## Inputs

- src/dkg/io.py
- src/dkg/tier1.py
- src/dkg/tier2.py
- src/dkg/tier3.py
- src/dkg/phases/phase1.py
- src/dkg/graph.py

## Outputs

- src/dkg/modes/xx.py
- src/dkg/graph.py
- tests/test_mode_xx.py

## Requirements

- Load single matrix via load_matrix(); use as both X and Y for symmetric analysis
- Compute Phase 1 marginal profiles; write tier0_marginals.parquet
- Run Tier 1 in xx mode (upper triangle only); run Tier 2 and Tier 3
- Build networkx.Graph: nodes = column names, edges = pairs with abs(pearson_r) >= config.graph_edge_threshold from tier1_df
- Run Louvain community detection via python-louvain package; add python-louvain via uv add
- Annotate each node with community_id; write graph.graphml via networkx.write_graphml
- Write communities.parquet with columns node, community_id, degree
- Document the graph_edge_threshold default and its effect in a module-level comment
