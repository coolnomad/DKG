# Task 002: RunConfig model, package entry point, and stub modules

## Objective

Define the RunConfig Pydantic model capturing all tunable parameters for a dkg run. Register the dkg CLI entry point in pyproject.toml. Create stub modules for each subsystem so all imports resolve cleanly before any implementation exists.

## Acceptance Criteria

- [ ] `from dkg.config import RunConfig` succeeds and `RunConfig()` instantiates with all defaults
- [ ] RunConfig fields include: mode, x_matrix_path, y_matrix_path, output_dir, tier1_pearson_threshold, tier1_spearman_threshold, top_k, n_jobs, graph_edge_threshold, spline_df, n_bins, n_boot, sample_frac, loo_influence_threshold, n_loo, seed
- [ ] All stub modules (dkg.io, dkg.tier1, dkg.tier2, dkg.tier3, dkg.phases.phase1 through phase10, dkg.modes.xy, dkg.modes.xx, dkg.modes.pair, dkg.graph, dkg.cli) import without error
- [ ] `python -m dkg --version` exits 0
- [ ] All default checks pass

## Requirements

- RunConfig uses pydantic BaseModel with all fields having sensible defaults
- mode: Literal["xy", "xx", "pair"] default "xy"
- x_matrix_path, y_matrix_path: str | None = None
- output_dir: str = "output"
- tier1_pearson_threshold: float = 0.2
- tier1_spearman_threshold: float = 0.2
- top_k: int = 1000
- n_jobs: int = -1
- graph_edge_threshold: float = 0.3
- spline_df: int = 3
- n_bins: int = 4
- n_boot: int = 200
- sample_frac: float = 0.80
- loo_influence_threshold: float = 0.5
- n_loo: int = 200 (defaults to min(n, n_loo) at runtime in Phase 1)
- seed: int = 1
- Stub modules contain only a module-level docstring and pass or stub function signatures
- Add entry point to pyproject.toml: dkg = "dkg.__main__:main"
- __main__.py prints version and exits 0 for --version; otherwise prints "not yet implemented" and exits 1
